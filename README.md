# near-agent

Autonomous agent for [market.near.ai](https://market.near.ai) вҖ” discovers jobs, evaluates fit, bids intelligently, executes work, and submits deliverables.

## What It Does

`near-agent` operates the full marketplace lifecycle autonomously:

1. **Discovers** open jobs via the market.near.ai API
2. **Evaluates** each job using LLM-based scoring (fit, effort, approach)
3. **Bids** on best-match jobs with tailored proposals
4. **Executes** awarded work using Claude (code, research, writing, analysis)
5. **Submits** deliverables with SHA-256 verification
6. **Communicates** with job creators via assignment messages
7. **Logs** every decision into structured demo transcripts

## Quick Start

```bash
# Clone and install
git clone https://github.com/ivinco/near-agent.git
cd near-agent
python3 -m venv .venv
.venv/bin/pip install -e .

# Configure
cp config.toml.example config.toml
# Edit config.toml with your market.near.ai API key

# Set Anthropic API key
export ANTHROPIC_API_KEY="sk-ant-..."

# Run one cycle
.venv/bin/near-agent run --cycles 1

# Run continuously
.venv/bin/near-agent run --continuous

# Check status
.venv/bin/near-agent status

# View active bids
.venv/bin/near-agent bids
```

## Configuration

```toml
[market]
base_url = "https://market.near.ai"
api_key = "sk_live_..."        # Your market.near.ai API key
agent_id = "your-agent-uuid"
handle = "your_handle"

[agent]
skills = ["python", "typescript", "api-development", "code-review"]
min_budget = 1.0               # Minimum job budget to consider
max_bid = 10.0                 # Maximum bid amount
max_concurrent_jobs = 3        # Max jobs to work simultaneously
poll_interval_seconds = 60     # Seconds between discovery cycles
bid_strategy = "competitive"   # competitive | premium | budget

[llm]
model = "claude-sonnet-4-20250514"
max_tokens = 4096
```

## How It Works

### Job Evaluation (LLM-Scored)

Each job is evaluated against the agent's skill profile:
- **Fit score (0-100):** How well the job matches agent capabilities
- **Can complete:** Boolean вҖ” does the agent have the tools to deliver?
- **Estimated hours:** LLM-assessed effort
- **Proposed approach:** Brief strategy for completing the work

Jobs scoring 50+ are eligible for bidding. Top 3 per cycle receive bids.

### Bidding Strategy

Proposals are generated by Claude, tailored to each job:
- References specific requirements from the description
- Explains the planned approach
- Sets competitive pricing (at or slightly below budget)
- Provides realistic delivery estimates

### Work Execution

When a bid is awarded, the agent:
1. Fetches the full job details
2. Passes requirements to Claude for deliverable generation
3. Produces complete work output (code, reports, analysis)
4. Computes SHA-256 hash for verification
5. Submits via the marketplace API

### Safety

- Budget ceiling per bid (configurable)
- Skips jobs below minimum budget threshold
- Filters out test/spam listings
- Graceful error handling with retries
